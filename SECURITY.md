# Security Policy

Проект обрабатывает платежи и персональные данные подписчиков — сообщения об
уязвимостях очень ценны.

## Как сообщить об уязвимости

**Не создавайте публичный issue.** Вместо этого:

- Откройте приватный отчёт: **Security → Report a vulnerability** на странице
  репозитория (GitHub Private Vulnerability Reporting), или
- напишите владельцу репозитория напрямую (контакты в профиле GitHub).

Опишите: затронутый компонент (бот / admin API / cabinet API / вебхуки платежей),
шаги воспроизведения и потенциальное влияние. Ответим как можно быстрее,
исправления критичных проблем — приоритет №1.

## Что считается уязвимостью

Обход аутентификации (admin JWT, `initData` мини-аппы), подделка платёжных
вебхуков, утечка секретов/ключей, SQL-инъекции, повышение привилегий, IDOR
в API. Поддерживается только актуальная версия `main`.
