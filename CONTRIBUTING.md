# Как поучаствовать

Спасибо, что хотите помочь! Короткие правила:

## Баги и идеи

- 🐛 Нашли баг → [заведите issue](../../issues/new/choose) по шаблону: шаги, окружение, логи.
- 💡 Есть идея → issue «Идея / фича» или обсуждение в [чате сообщества](https://t.me/vpnhub_community).
- Перед созданием проверьте, нет ли уже похожего issue.

## Pull Request'ы

1. Форкните репозиторий и создайте ветку от `main`.
2. Локальная разработка описана в [README](README.md#-установка-одной-командой)
   (раздел «Ручной запуск») и [docs/setup.md](docs/setup.md).
3. Перед отправкой прогоните гейт:
   ```bash
   make check   # ruff + mypy strict + pytest
   ```
   Для админ-SPA: `npm run build --prefix admin` должен собираться без ошибок.
4. Правила кода — [ARCHITECTURE.md](ARCHITECTURE.md) и `docs/context/07-gotchas.md`:
   слои не нарушать (сервисы не импортируют aiogram/FastAPI/httpx напрямую),
   деньги — целые minor-units, даты — UTC-aware, async везде, типы обязательны.
5. Новая модель БД или платёжный провайдер — по рецептам в [docs/recipes/](docs/recipes/).
6. Один PR — одно логическое изменение. Тесты на новое поведение обязательны.

## Безопасность

Уязвимости — **не** в публичные issue, см. [SECURITY.md](SECURITY.md).
